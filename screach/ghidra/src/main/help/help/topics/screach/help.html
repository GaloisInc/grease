Screach

The Screach plugin drives the Screach reachability analyzer from within Ghidra. You mark an entry (source) address and a target address in the Listing, run Screach, and view its results in a docked window. Screach determines whether the target is reachable from the entry using symbolic execution.

Setting the Entry Address

Right-click an instruction in the Listing and choose Screach → Set as Screach Entry (Source) Address.

Setting the Target Address

Right-click an instruction and choose Screach → Set as Screach Target Address. This is the location Screach will try to reach.

Adding Avoid Addresses

Right-click and choose Screach → Add Screach Avoid Address to tell Screach to schedule any path through that instruction last (--avoid-addr). You may add several.

Clearing Avoid Addresses

The entry and target addresses are each replaced whenever you set a new one, but avoid addresses accumulate. To discard all of them, choose Screach → Clear Screach Avoid Addresses from the menu bar. (The item is disabled when no avoid addresses are set.)

Running Screach

Choose Screach → Run Screach from the menu bar, or press Ctrl+Alt+R (rebindable in the Key Bindings options). Both an entry and a target address must be set. Screach runs as an external process off the UI thread and can be cancelled from the task dialog.

Results Window

The Screach window streams the command output and shows a colored banner: green when the target was reached, amber when it was not, and red on error. Toolbar buttons let you clear the log, navigate to the last address reported, and locate the coverage file.

Options

Configure the plugin under Edit → Tool Options → Screach:

  • Use Docker – run Screach via a Docker image instead of a local binary.
  • Binary Path – path to the screach executable (when Use Docker is off).
  • Docker Image / Docker Run Args – image name and extra docker run arguments (when Use Docker is on). The binary's directory is mounted and used as the working directory automatically.
  • Extra CLI Args – arguments appended to every invocation.
  • Solver – value for --solver (bitwuzla, cvc4, cvc5, yices, z3).
  • Dump Coverage – pass --dump-coverage to produce a .greasecov file. Load it in grease-cartographer via Tools → Code Coverage → Load Code Coverage File(s)…
